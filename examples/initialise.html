Initialising ZEngine

This is an easy one! You can initialise ZEngine a few different ways, but the simplest is to just pass a query selector that belongs to a canvas element:

ZEngine.Initialise({Canvas: "#ZEngineCanvas"});

Your HTML should have this somewhere so ZEngine knows where to put it:

<canvas id="ZEngineCanvas"></canvas>

Another way to initialise ZEngine is to pass it a parent element:

ZEngine.Initialise({Parent: "#ZEngineParent"});

This would look for the #ZEngineParent element and "inject" a new canvas into that area.

By default ZEngine has a strange canvas size.. this is because in almost every case you will want to set your own anyway, and we wouldn't want to assume what size you like best! So to choose the size you can use the aptly named Width and Height options:

ZEngine.Initialise({Canvas: "#ZEngineCanvas", Width: 800, Height: 600});

Simple huh! Lastly, another very handy thing about initialising ZEngine is you do not need to worry about the DOM being loaded, because it will check this first anyway, another handy use of the Initialise method is you can pass a second parameter that will only be called after the DOM is ready, so you might want to put all your ZEngine related code in here for a number of reasons. Here is a typical initalisation in production:

ZEngine.Initialise({Path: "ZEngine", FPS: 60, Canvas: "#MyCanvas"}, () => {
    // All your ZEngine code, and maybe more?
});

Notice two more options here, Path will need to be a path from the calling page to the directory that ZEngine.js is stored in, by default ZEngine will assume the path to just be in the "ZEngine" directory in the same location as the page being used.

Secondly we have the FPS option, by default this is set to 60, which will mean every 1000 JavaScript ticks Update methods will be called 60 times.
